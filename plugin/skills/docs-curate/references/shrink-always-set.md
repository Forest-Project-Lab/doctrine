# 常時集合の縮小（§3.9）

常時注入する集合が上限を超えたら縮める。上限は `inject-contract.py` が持ち、超過の合図を出す。縮小の作業はこの技能が持つ。

## 上限との対

`inject-contract.py` は SessionStart で常時集合を描き、上限を守る。上限を超えるなら、その旨と `docs-curate` の起動を促す合図を出す。この技能はその合図を受けて縮める。

## 縮め方

1. 決定事実を一本化する。同じ主題の複数の DECIDED 項目を一つの正本へまとめる。
2. 期限切れの `review_by` 項目を片づける。再検討の期限を過ぎた項目を整理する。期限切れを理由に方針を黙って解除しない。再検討は人間が ADR で決める。
3. 廃止した文書の本文は常時集合から外す。事実だけ DECIDED の対の記録に残す。

## 何を残すか

何を残すかの最終判断は人間に依る（§7）。機械は候補を示すが、どの事実を常時集合に残すかは人間が決める。
