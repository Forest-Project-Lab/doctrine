# 逆参照の確認

降格の前に、現行の依存元がゼロかを確かめる。これは §3.8 の不変条件である。

## `dep-graph.py` を逆方向で呼ぶ

`dep-graph.py --reverse-refs <id>` は、その文書へ `depends_on` で依存する現行文書を返す。返りが空なら降格してよい。空でなければ降格しない。

## 依存元の付け替え

依存が残るなら、降格の前に、同じ変更の中で依存元を後継へ付け替える。

1. 後継文書の id を決める。
2. 依存元の `depends_on` を後継の id へ書き換える。
3. もう一度 `dep-graph.py` を逆方向で呼び、現行の依存元がゼロを確かめる。
4. ゼロになってから降格する。

削除安全ガードがこの不変条件を守る。迂回しない。Bash の削除（`rm`／`git rm`／`mv`）では文脈注入がモデルへ届かないため、この技能が自ら逆参照を確かめてから提案する（§7）。
