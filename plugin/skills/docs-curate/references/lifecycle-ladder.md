# 状態の階段（§3.8）

文書は一段ずつ降ろす。逆向きに戻すには根拠を要する。

## 五つの段

現行 → 廃止 → アーカイブ → git だけ → 抹消。

- 現行: いま効いている版。
- 廃止: 方針を撤回した状態。本文は LLM へ渡さない。事実だけ DECIDED の対の記録に残す。
- 置換: 後継で入れ替えた状態。`superseded_by` で後継を指す。
- アーカイブ: 証跡として倉庫へ退避した状態。不変性ガードが編集を拒否する。
- git だけ・抹消: 倉庫からも外し、最後に消す。再現できる一時文書だけが対象。

## 戻せる範囲

一段ずつ降ろせば後から引ける。まとめて消すと戻せない。だから一片ずつ進める。

## 後から引ける記録

降格はすべて後継 id とともに記録する。後から「何を、なぜ、どこへ」降ろしたかを引けるようにする。`superseded_by` と DECIDED の対の記録がこの索引になる。
