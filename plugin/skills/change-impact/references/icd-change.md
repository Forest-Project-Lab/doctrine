# ICD 変更とドメイン間の合意（§3.5）

ICD の中身が変わるときは、依存する全ドメインの合意が要る。`[R7]`

## なぜ合意が要るか

ICD は、そのドメインが外へ公開する接点である。他ドメインは ICD だけを `depends_on` できる（§3.6）。ICD が変わると、依存する全ドメインに影響する。だから、変える前に合意を取る。

## 手順

1. `dep-graph.py` を逆向きで走らせ、その ICD に依存する文書とドメインを列挙する。
2. ドメイン跨ぎの境界を分類する（`change-impact` ステップ5）。
3. 依存する全ドメインに変更を示し、合意を取る。
4. 合意の後に、`ADR` で決め、ICD を更新する（順序は変更フローに従う）。

## 境界を越えない設計

ICD を変えない限り、内部の変更は境界を越えない。内部が古くなっても、ドメインごと差し替えられる（§3.6）。だから、できるだけ内部で閉じ、ICD の変更は最小にする。
