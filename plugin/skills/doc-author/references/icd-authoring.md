# ICD の書き方（§3.5／付録A）

各ドメインは、外から依存してよいものを `<domain>/ICD.md`（型 `ICD`）一文書に集約して公開する。`[R7]`

## 宣言する三つ

1. **公開する用語**: そのドメインが所有し、他ドメインが使ってよい語。用語辞書の該当分への参照。
2. **正本である事実**: そのドメインが唯一の正本である事実。`canonical_for` のトピックと一致させる。
3. **データ契約**: 他ドメインが依存してよい `入出力`・スキーマ・更新頻度。

## 書かないこと

ICD に内部実装は書かない。内部の検討も書かない（§3.2 の「入れない」）。

## テンプレート

`templates/icd.md.tmpl`（付録A）をそのまま使う。`status: current`、`canonical_for` を持つ。`llm_context` 行は無く、登録簿の既定 `task` が効く。

## 変更時の合意

ICD の中身が変わるときは、依存する全ドメインの合意が要る。利用者にこれを伝える。ICD を変える前に、`dep-graph.py` の逆方向で、その ICD に依存する文書を数える。

## Write 経路へ寄せる

ICD を含むドメイン間の依存を持つ文書は、Edit や MultiEdit より Write 経路で書く。そうすればガードが実行前に違反を拒否できる（§3.8／§7）。
