# 突き合わせの手順（DECIDED・WATCH）

提案された変更を決定事実（DECIDED）と戻してはならない一覧（WATCH）に突き合わせる手順を記す。

## DECIDED との突き合わせ

DECIDED 正本（`_system/decided-facts.md`）は、決定事実・決定日・根拠 ADR・`review_by` を持つ。

1. 変更が触れる主題を読む。
2. 同じ主題の決定事実があるかを探す。
3. 変更がその決定事実に反するなら指摘する。決定は ADR でしか変えられないため、覆すには新しい ADR を要ると伝える。
4. 決定事実の `review_by` を見る。過ぎているなら、再検討の時期だと記す。期限切れの事実に黙って頼らない。

## WATCH との突き合わせ

WATCH 正本は、撤回日・根拠・`review_by` を持つ。戻してはならない方針を記録する。

1. 変更が再び持ち込む方針・依存・状態を特定する。
2. WATCH に一致する項目があれば、元の根拠とともに退行として指摘する。
3. WATCH 項目の `review_by` を見る。過ぎていても、撤回を黙って解除しない。再検討は人間が ADR で決める。

## `review_by` の扱い

`review_by` を過ぎた DECIDED・WATCH は古い事実である。期限切れを理由に自動で解除しない。監査が `review_by` 超過を一覧化し、`docs-curate` と `regression-guard` がそれを受け取る。どちらも期限切れの事実を黙って信頼しない。
