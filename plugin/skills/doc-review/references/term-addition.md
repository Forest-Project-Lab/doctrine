# 承認語の追加（§1）

承認語を増やす、または意味を変えるときは、`ADR` で決め、用語辞書（§1）の表を更新する。各文書はこの表を二重定義しない。`[R6][R8]`

## 流れ

1. 新しい承認語が要る兆候を見つける。同じ語が二つの意味を持つなら、ドメインを分ける合図でもある（§3.1）。
2. `term-extract.py`（`c-TF-IDF`）の候補を証跡にする。これはドメイン特徴語の素案を出すだけで、確定はしない。
3. `ADR` を起こして決める（背景・却下した選択肢・決定・帰結、1文書1決定）。
4. 用語辞書の正本（`docs/_system/glossary.md`）の表に一行足す。承認語・唯一の意味・禁止する同義語。
5. 自分で承認しない。`ADR` と辞書更新を必須にする。

## 新しいカルク

新しい訳語臭を見つけたら、別ファイルを作らず、§1 のカルク表に一行足す。使わない表現・直す表現・なぞった英語の三列で書く。
