# 逆翻訳テル（§1）

判定は逆翻訳テル。怪しい箇所を英語に戻し、英語の慣用句・固定表現にそのまま戻れば、それはカルクである。直すときは英語を捨て、日本語で選び直す。`[R10]`

## 一覧のカルク（リンタが照合する）

§1 の禁止表現の表は、リンタ（用語チェッカー）が機械的に照合する。代表例を、なぞった英語と直しで示す（実例の語そのものは `code` で囲んで、点検が自分自身を弾かないようにする）。

- `同じページにいる` → 認識を揃える（`on the same page`）。
- `針を動かす` → 効果を出す（`move the needle`）。
- `深く潜る` → 詳しく見る（`deep dive`）。
- `一日の終わりには` → 結局（`at the end of the day`）。

## 一語訳の罠（辞書の一番目を疑う）

- `status` → 位置づけ・区分。
- `native` → 標準で・組み込みで。
- `robust` → 壊れにくい。
- `leverage` → 活かす。

## 擬陽性を避ける

- 定着した借用語（`データ`・`リスク`）はカルクではない。
- 普通の否定文はカルクではない。
- 一覧に無い訳語臭だけを逆翻訳テルで判定する。一覧の分はリンタに任せる。

## 一覧に無いカルク

§1 の一覧に無い訳語臭は、このスキルが逆翻訳テルで拾う。新しいカルクは、別ファイルを作らず、§1 の表に一行足す。`ADR` と用語辞書の更新を必須にする。
