# 文章の規則（§1）

doc-review が点検する。`[R10]`

## 規則

- 一文一義。係り受けが二通りに取れる文は短く割る。
- サ変名詞＋「を行う」を動詞にする（「初期化を行う」→「初期化する」）。
- 「することができる」→「できる」。
- 専門語・略語は初出で定義する。
- 抽象語は具体語・数値・固有名に置き換える。
- 同じ概念は同じ語で書く（用語辞書、§1）。
- テストできる形で書く（§2・ADR-020）。REQ・SPEC の記述が「〜したい」「柔軟に」「適切に」のような、合否を確かめられない願望に留まっていたら弾く。確かめ方（入力と期待する結果、または受入基準）を書けるまで具体化させる。この判定は意味の判断であり、監査（docs-audit）は照合しない。ここが唯一の検出点である。

## 当て方

1. 用語チェッカーの機械的な指摘を先に集める。
2. 上の規則を一つずつ当てる。直せる箇所は直しの案を添える。
3. 指摘は助言として出す。硬い拒否はしない。

## 用語辞書を二重定義しない

語彙の正本は §1 の用語辞書だけである。このスキルは語彙を定義し直さない。強制するだけである（`R6`）。承認語を増やすときは §1 の流れに従う。
