# Hook 設定の取り込みの前提

## セッション開始時にだけ取り込まれる

Hook の設定はセッション開始時に取り込まれ、その場では効かない（§5 運用上の前提・§7）。ガードやリンタを変えたら、新しいセッションで効く。

## 運用上の含み

- 設定を変えた直後の同じセッションでは、古い設定が動く。利用者に新しいセッションを始めるよう促す。
- このため、ガードの変更は計画して行い、変更後に動作を確かめる。
- この前提は構造で閉じない。Claude Code の Hook の仕様に依る（§7・付録C）。

## 報告に含める一文

設定を置いたあと、報告にこの一文を必ず添える。「Hook の設定はセッション開始時に取り込まれる。ガードを変えたら新しいセッションで効く」。
