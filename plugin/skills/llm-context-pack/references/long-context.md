# 長い文脈の扱い（§3.9 の根拠）

長い入力は、それ自体が成功率を下げる。§3.9 はこれに抗う三つの規則を置く。

## なぜ端へ置くか

長い入力では、冒頭と末尾が注意を引きやすく、中ほどが薄れやすい。最も重要な文書を冒頭と末尾の両方に置けば、重要な事実が中ほどで埋もれない。

## なぜ復唱するか

長い本文の前に、モデルへ要点を先に書かせる。要点を先に言語化すると、長い文脈での劣化に抗える。

## なぜ上限を設けるか

入力が長いこと自体が成功率を下げる。だから最少被覆集合に絞り、`never` を除外し、上限を守る。最適な上限の値は構造では定まらない。運用と受入基準で定める（§7）。
