# 最少被覆集合の計算

タスクの要求を満たす最少の文書集合を求める手順を記す。`collect-context.py` がこの計算を持つ。

## 被覆の計算

1. タスクが触れる要求（REQ）を列挙する。
2. 各要求を被覆する文書を `dep-graph.py` の `depends_on` グラフから引く。
3. 要求の集合を被覆する最少の文書を貪欲に選ぶ。
4. 依存の閉包を引き、被覆に要る ICD を加える（ICD は `never` ではない）。

## 縮小の順序（§3.9）

縮める順序は決まっている。

1. まず取りこぼしを減らす。要求を被覆しない文書が混ざっていないかを見る。
2. 次に重複を除く。同じ事実を別の文書から二重に入れていないかを見る。逆向きに刈り取る走査で、被覆を保ったまま重複を外す。

## 除外の規則

- `llm_context: never` の文書（RESEARCH・ARCHIVE）は、被覆の計算より前にすべて除外する。本文は LLM へ渡らない（R5）。
- 関係しない文書を足さない。被覆されない要求は `uncovered` として報告する。
