# 保証限界の明示（各成果物が予防・検出・委ねるを書く）

## 要求文

配布する各成果物は、何を予防し、何を検出し、何を人間に委ねるかを明示しなければならない `[R9]`。プラグインは「100%の予防」を約束しない。利用者がそれを読み取れる形で書く。こう書くことで、ガードやリンタが拒否しない範囲を、運用が誤って安全と見なすのを防ぐ。テストできる形では、README（案内ファイル）と各技能ファイルの SKILL（技能を記す `SKILL.md`）が `## 保証限界` 節を持つことで確認する。

## 優先度

高

## 受入基準参照

TEST-020（README と各成果物が `## 保証限界` の三区分を持つことを確認する。受入は §6 メタ条件 R9 行に対応）。

## 出所

設計書 §7「保証限界」。要求 `R9`。
