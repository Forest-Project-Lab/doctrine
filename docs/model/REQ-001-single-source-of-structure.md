# 構造規則とメタデータ様式を単一の正本として定義する

## 要求文

型の登録簿（18 型、既定 `status`、既定 `llm_context`、置き場所）、`status` の統制語彙、フロントマターの必須キーと様式を、コード内に一度だけ正本化し、機械可読に引けること。ガード・リンタ・監査・graph などほかのスクリプトは、これらの規則を二重定義せず、この正本だけを引くこと。[R2][R3][R6][R8]

フロントマター解析は、内容がどうであっても例外を投げないこと。編集途中の半端な内容でも、読める所まで解析した結果と構造化エラーを返すこと。こうすることで、毎ターン起動する Hook（イベントの発生時に走る処理）が落ちない。

## 優先度

高

## 受入基準参照

TEST-001（登録簿契約が DATA-001 と一致すること、および許可表）、TEST-002（解析行列と、どの入力でも例外を投げないこと）。

## 出所

spec §3.2（型登録簿）、§3.3（`status` 統制語彙）、§3.4（メタデータ・スキーマ）、§4.3（標準ライブラリだけで書く最小パーサ）。
