# 構造規則の単一正本化

ADR（Architecture Decision Record、決定の記録）。本文書は一つの決定だけを扱う。C2とは、凍結した契約の整合を見る判断項目の番号をいう。[R6][R8]

## 背景

型登録簿、`status` 統制語彙、必須キーは、ガード・リンタ・監査・graph・context の各スクリプトが使う。これらを各スクリプトに書くと規則が分散し、変更したときに食い違う。spec §3 は、コードに規則を二重定義しないと定めている。

## 却下した選択肢

- 各スクリプトに規則を持たせる案。規則が二重定義になり、変更したときに食い違う。却下した。
- 設定ファイル（外部 YAML）に置く案。外部への依存と、読み込みに失敗する経路が増える。動きも決定的でなくなる。却下した。

## 決定

構造規則を共有モジュール `scripts/_registry.py` に一度だけ書き、ほかのスクリプトはこれを引く。中身は純データと純関数だけとし、標準ライブラリだけで書く。

## 帰結

規則を変えるときは一箇所だけ直せばよい。集合やリストは複製を返し、呼び出し側が登録簿そのものを書き換えられないようにする。`domain_of` はここに置かず、graph に委ねる。本決定は、DECIDED-001 の確定事実（registry を単一の正本とする）の根拠となる。
