# 型付き文書を正しい場所と様式で作り初期化は非破壊・最小に保つ

## 要求文

型付き文書（ICD を含む）を、§3.2 の置き場所と §3.4 の様式で作れる。`id` はファイル名と一致させる。ドメインのフォルダと各層は、その型の文書を最初に書くときに、そのつど生成する。初期化は既存を壊さず、`_system` の最小集合だけを置く。中身の無い足場を先に作らない。`[R1][R8]`

## 優先度

高

## 受入基準参照

§6 の R1 行（Overview 投影が現行の文書をすべて含む）と R8 行（最小性）。TEST-015・TEST-017 で確認する。

## 出所

spec §3.7（配置、そのつどの生成、中身の無い足場を先に作らないこと）・§3.4（メタデータ）・§4.4（段階的な導入）。
