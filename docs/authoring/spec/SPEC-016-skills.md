# skills（7技能を一仕様で）

7つの技能（Skill。人の判断を支援する成果物）を、一つの仕様でまとめて定める。各技能は、機械で割り切れる処理を scripts と登録簿に任せ、人の判断を要する部分だけを担う。`[R1][R3][R4][R5][R6][R7][R8][R9][R10]`

## 入出力

入力は利用者の依頼である。各技能は、`description`・役割・任せる先を持つ。`description` は三人称で書く起動文で、技能の起動を促す語句と、利用者が実際に使う語句の両方を含める。7つの技能は次のとおり。

- `docs-system-init`（R1・R8）: `_system` に最小限のファイルと案内を置き、ガードとリンタを組み込む。既存は壊さない。実際のファイル生成は `scaffold.py` に任せる。
- `doc-author`（R1・R3・R6・R7）: 型付き文書（ICD を含む）を作り、更新する。正本である登録簿（ICD-001）に従う。
- `doc-review`（R6・R10）: 文章の規範と位置づけを点検し、その傍らで用語チェッカー（リンタ ICD-004）を走らせる。一覧に無いカルクは逆翻訳テルで判定する。
- `change-impact`（R3・R4）: 変更の手順を14段階で進める。依存をたどって影響を列挙し、決められた更新順を守る。
- `regression-guard`（R5）: 一度廃止したものの復活や、撤回した方針の再採用を防ぐ。DECIDED・WATCH と突き合わせる。
- `llm-context-pack`（R5）: タスクごとに、必要な文書を過不足なく集めたまとまりを作る。`llm_context:never` の文書は除き、各文書の出所を示す（`collect-context.py`、ICD-006）。
- `docs-curate`（R8）: 点検・統合・降格・削除を一つずつ進める。どこからも依存されていないかを確かめ、投影を作り直す。

## 制約

各技能の本文は500行未満に収め、詳細は `references/` に分ける。`description` は三人称で書く。機械で割り切れる処理は `scripts/` に任せる。技能は7つに固定し、ICD 専用の技能は作らない。集約と削除は一つの活動とみなし、`docs-curate` にまとめる。各技能は `## 保証限界` 節を持ち、何を予防し、何を検出し、何を人に委ねるかを書く。使う語は承認語に限る。

## エラー時挙動

Level 2 の縮小構成では、`change-impact`・`docs-curate` と、監査を前提とする `doc-review`・`regression-guard` の定例の動きが、機能を一部落としつつ穏やかに減る。異常終了はせず、何ができなくなるか、それにはどの Level が要るかを述べる。

## 受入基準

技能がちょうど7つあること。各 `description` が §7 の文言と一字一句一致し、三人称で500行未満であること。各技能が `## 保証限界` 節と、予防・検出・委ねるの三層を持つこと。各技能の本文が用語チェッカーで誤りを出さないこと。以上を TEST-016 が確認する。
