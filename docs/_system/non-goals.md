# 横断のやらないこと（5項）

本文書は、体系があえて引き受けない範囲の正本である。どこまでを保証しないかを明示する（[R8]・[R9]）。各項は、対象を具体に書き、理由を添える。

## やらないこと

1. 意味的に重複する文書を、最終的に同じかどうか判断することはしない。リンタ（文書を機械点検するスクリプト）と監査は、語彙の近い重複に印を付けるだけにとどめる。統合してよいかは人が決める。
2. カルク（英語の比喩・構文・一語訳をなぞった日本語）をすべて見つけ出すことはしない。用語チェッカーが照合するのは、一覧に載った表現だけである。
3. 違反を100%防ぐことはしない。ガード（違反を機械的に拒否するHook）は防ぐための最初の関門であり、ここを逃れた違反は監査が後から拾う。
4. ガードを通さずに書かれた文書までは防げない。Hookが起動しない経路で書かれた文書は止められず、全件監査が後から補う。
5. ターン（一回のやり取り）ごとに全件を走査することはしない。リンタは編集された一つの文書だけを点検し、全件の点検は監査に委ねる。

## 理由

1. 意味が同じかどうかは文脈に依存する。機械に白黒を付けさせると、擬陽性も取りこぼしも生む。判断を人に残すほうが安全である。
2. 言い回しは限りなくあり、すべてを網羅することはできない。確実なものだけを一覧で弾き、残りは doc-review の逆翻訳テルに委ねる。
3. すべての書き込み経路を塞ぐと、運用が止まる。予防・検出・委ねるの三層に分けるほうが壊れにくい。
4. Hookは登録された経路でしか起動しない。これは仕組み上どうしても残る穴であり、監査の全件走査で埋める設計とした。
5. ターンごとに全件を走査すると、応答が遅くなる。編集された一文書の点検と、定例での全件監査に役割を分ける。
