# 投影を正本から描画し直せる派生表示に限り、刊行物は投影一覧に含めない

## 背景

頑健性評価の検証で、§3.9 の投影（正本から描画し直せる派生表示）と、外部ツールのデータから作る派生表示（ガントチャート・バーンダウンチャート・ダッシュボード）の混同が系統的に起きた。投影と判定された82件のうち、複数の調査系統で後者を正す修正が入った。定義は §3.9 に明文であるのに誤読が繰り返された。あわせて、複数の現行文書から組み立てる刊行物（FAQ（よくある質問集）・リリースノート・操作手引）は §3.9 の投影一覧になく、扱いが未定だった。[R8]

## 却下した選択肢

- 派生表示をすべて投影と呼ぶ。正本が体系の外にあるものは描画し直せず、投影のずれ検出も効かない。語の意味が薄まり、一語一義（R6）に反する。
- 刊行物を投影一覧へ足す。render-projection の拡張が要り、必要が出る前の作り込みになる（R8）。

## 決定

投影は、体系の正本（登録簿・フロントマター・現行文書の本文）から描画し直せる派生表示に限る。正本が体系の外にある派生表示は投影と呼ばず、docs/ へ写さない。刊行物（複数の現行文書から組み立てる配布物）は投影一覧に含めず、体系の外の配布物として正本から組み立てる。配布物を正本にしない。投影の種類を足すときは ADR で決める。

## 帰結

投影の語が締まり、外部正本の派生表示の写し込みを点検の対象にできる。刊行物の運びは出典 S1000D の「刊行物は組み立てる」と揃う。保証限界: 刊行物と正本のずれは、投影のずれ検出の外にある（組み立てのたびに正本から作り直すことで抑える）。
