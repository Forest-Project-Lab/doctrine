# 統治の対象を知識と決定の層に限る

## 背景

頑健性評価は、ビジネスのアウトプット205件のうち約半分を統治の対象外と判定した。業務トランザクション（請求・受発注・仕訳）、外部ツールが正本の動的データ（チケット・台帳・計測値）、署名済みの法的文書は、doctrine_docs/ へ写すと正本が二つになり（R8違反）、鮮度でも腐る（R2違反）。だがこの境界はどこにも明文でなく、「何でも doctrine_docs/ へ写す」誤用を予防できなかった。[R8][R9]

## 却下した選択肢

- 境界を明文にせず、各導入先の判断に任せる。誤用を予防できず、常時集合の肥大（R5）を招く。
- 業務トランザクションも写せるよう型を増やす。外部の正本と二重になり、「各事実の正本を一つだけ保つ」思想に反する。

## 決定

統治の対象を「知識と決定の層」（仕様・決定・確定事実・調査・手順）に限る。業務トランザクション、外部ツールが正本の動的データ、署名済みの法的文書は統治しない。許すのは、要点の転記（DECIDED・ADR へ）と出所の参照だけとする。この境界を NONGOAL と §7 に書く。

## 帰結

導入先は、外部正本のデータを doctrine_docs/ へ写す前に、この境界で判断できる。監査は、影文書の検出でこの境界の破れを事後に拾う。保証限界: 境界の内外の判定そのものは人が下す（機械では「知識か作業データか」を判別できない）。
