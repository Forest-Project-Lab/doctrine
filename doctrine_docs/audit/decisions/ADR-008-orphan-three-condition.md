# 孤児を三条件の連言で定義する

ADR（設計判断の記録）。一文書一決定。`[R8]`

## 背景

監査は、削除してよい候補として孤児を一覧にする。ここで「どこからも依存されていない」ことだけを孤児の条件にすると、まだ現役の入口や、作ったばかりの文書まで誤って削除候補に入ってしまう。

## 却下した選択肢

- 依存されていないことだけを条件にする案: 入口（投影や常時文脈の文書）と新規の文書まで孤児と誤判定するため却下した。
- 「依存されていない」かつ「陳腐化している」の二条件にする案: 一度消すと作り直せない正本まで削除候補に入れてしまい、取り返しのつかない損失を招くため却下した。

## 決定

孤児を、次の三条件すべてを満たす文書と定義する。(1) どの現行文書からも依存されていない、(2) 陳腐化している（`updated` が古い、または `review_by` の期限を超過している）、(3) 再現可能である。再現可能かどうかは `_is_reproducible` が三つの分岐（`type`==RESEARCH、`llm_context`==never、`reproducible: true`）で判定する。ICD・投影・`llm_context`==always の文書は、入口や常時文脈に当たるため孤児にしない。

## 帰結

削除候補が安全側に絞り込まれる。作り直せない正本は、陳腐化しても孤児にはならず、消しても安全な状態が保たれる。この判定は毎回同じ結果になり、`docs-audit.py` の `_check_orphan` に実装されている。
