# 作成・初期化の設計判断（7技能固定・遅延生成・テンプレが語彙符号化）

本 ADR（Decision Record。背景・却下案・決定・帰結を、1文書につき1決定の形で記す決定の記録）は、authoring の三つの設計判断を一つの決定としてまとめる。

## 背景

人の判断を技能で支援するにあたり、技能をいくつ置くか、初期化でどれだけの足場を置くか、語彙辞書をどこに置くかを決める必要がある。同時に、中身の無い型や足場を先に作らないという最小性（R8）を守りたい。

## 却下した選択肢

- 技能を機能ごとに細かく分ける案（ICD 専用の技能や削除専用の技能を別に置く）。役割が重なって保守の手間が増えるため、却下した。
- 初期化の時点で、ドメインのフォルダと各層をすべて先に置く案。中身の無い足場が孤児を生むため、却下した。
- 各技能・各スクリプトが §1 の辞書をそれぞれ別に持つ案。二重に定義した辞書どうしが食い違うため、却下した。

## 決定

技能は7つに固定する。集約と削除は `docs-curate` にまとめ、ICD は `doc-author` が書く。ドメインのフォルダと各層は、その型の文書を最初に書くときに `doc-author` がそのつど生成する。§1 の語彙は、テンプレート（`glossary.md.tmpl`）が一度だけ書き写し、用語チェッカーはそれを運用版として読む。

## 帰結

技能の役割が重ならず、保守する対象が小さくなる。初期化が既存を壊さず最小限に収まり、中身の無い足場が孤児を生まない。辞書の正本が一つに定まり、食い違いが起きない。確定した事実は DECIDED-001 が持つ。
