# 用語チェッカーは never 文脈の RESEARCH・ARCHIVE を点検しない

## 背景

RESEARCH と ARCHIVE は `llm_context: never` で常時文脈に入らず、外部由来の語彙（規格名・法令名・外部成果物名）を宿す型である。頑健性評価の報告（RESEARCH-001、584件の外部成果物カタログ）を統治木へ取り込むと、用語チェッカーが禁止同義語・未定義語で約170件の助言を出した。その大半は外部成果物の正式名に埋め込まれた禁止同義語（`要件定義書` など、68件中61件が名の一部）と外部の頭字語であり、地の文の違反ではない。R6 用語統一の便益は「文脈に載る文書の語彙を揃える」ことにあり、never 型はそもそも文脈に載らない。[R6][R8]

## 却下した選択肢

- 外部の頭字語を全て固有名として辞書へ登録する（ADR-018 の経路）。辞書は `llm_context: always` で常時注入されるため、150語規模の登録は注入上限（DECIDED-001 事実6）を圧迫する。擬陽性を消すために常時文脈を肥らせるのは本末転倒である。
- 報告の地の文を機械的に置換する。`要件定義書` を「要求定義書」に変えるように実在の成果物名を壊し、事実性を損なう。ADR-017 が固有名と引用の原文を doc-review に委ねたのはこのためである。
- 助言を放置する。触るたびに約170件の擬陽性が鳴り、本当の違反が埋もれる。ADR-017 が擬陽性の害として却下した論法と同じである。

## 決定

用語チェッカー（`_termcheck.check`）は、`type` が RESEARCH または ARCHIVE の本文を、GLOSSARY 正本・投影と同じく点検そのものを飛ばす。飛ばす根拠は型であって、上書き後の実効 `llm_context` ではない（既存の GLOSSARY・投影の飛ばしと同じ扱い）。地の文では承認語を使う規範（ADR-017）は変えない。これら never 型の地の文・固有名・引用の判断は、従来どおり doc-review（人の判断層）に委ねる。

## 帰結

never 型の外部カタログを統治木へ取り込んでも、用語チェッカーは鳴らない。構造の点検（必須キー・置き場所・id とファイル名の整合・型と置き場所）は従来どおり働く。全件監査は用語を点検しないため、この変更で監査の挙動は変わらない。保証限界: RESEARCH/ARCHIVE の地の文に紛れた真の訳語臭は機械では拾わず、doc-review の定例に委ねる（ADR-017 と同じ委譲先）。
