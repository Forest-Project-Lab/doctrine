# 構造語彙を正本で定義済みと認め、doc-reviewを著述時の閉じた輪にする

## 背景

用語チェッカーの未定義語ヒューリスティックが、体系自身の型コード（`SPEC`・`REQ`・`TEST` など）と要求タグ（`[R番号]`）を未定義語として大量に指摘していた。これらは別の正本で既に一度だけ定義されている。型コードは登録簿（§3.2）、要求タグは §2 の要求である。辞書にも書けば二重定義になり、§4.3 の「辞書を二重定義しない」に反する。[R6][R8]

## 却下した選択肢

- 型コードを用語辞書に転記する。登録簿との二重定義になり、§4.3 に反する。語の定義の在処が二つになり、片方が古びる。
- 指摘を放置する。助言の精度が落ち、本当の未定義語が雑多な指摘に埋もれる。

## 決定

用語チェッカーは、登録簿（§3.2）の型コードと §2 の要求タグを定義済みと認め、未定義語として指摘しない（`_termcheck` が `_registry` を引く）。doc-review は著述・編集のたびに走り、判断的な指摘をそれぞれの定義の在処へ書き戻す。一覧外のカルクは用語辞書のカルク表へ、新しい承認語は ADR と用語辞書へ加える。doc-author を介さない手編集には、PostToolUse の `review-nudge.py` が doc-review を促す（助言のみ。`decision` は出さない）。[R6][R10]

## 帰結

構造語彙の指摘が消え、残る指摘は外部の略語と一時的な参照に絞られる（本コーパスで未定義語の指摘が 386 件から 58 件へ減った）。doc-review が判断の層を著述時の閉じた輪にし、語の定義の在処を一つに保つ。型コードの意味が変わるときは、登録簿を直せば用語チェッカーへ自動で伝わる。
