# 寄稿の方針

このリポジトリは、情報統治の単一仕様を公開する。寄稿を歓迎する。ただし、本体系の思想（文書は負債、正本は一つ、常時投入は最小）を寄稿自身も守ること。

## 文章規範

すべての文書は、仕様 §1（統制語彙と文章の規則）に従う。

- 一文一義、一段落一トピック。段落の先頭文で、その段落の役割（結論・根拠・例外）を示す。
- 推量と断定を分ける。未決を既決として書かない。
- 「100% 検出できる」「必ず解決できる」と書かない。条件付きで述べる。
- 同じ概念は同じ語で呼ぶ。用語は初出で定義する。
- LLM 的な空句を使わない（`重要なのは`・`不可欠`・`多角的`・`掘り下げる`・`〜において` など）。
- 冗長を避ける。同じ主張を繰り返さない。読者が補える中間段階を書かない。

## 単一の正本

同じ事実を二箇所に書かない。規則を変えるときは、仕様（`spec/`）でのみ変更する。テンプレートや文書に規則を二重定義しない。

## 変更の出し方

- Issue で、変更の目的と背景を述べる。
- Pull Request は、一つの主題に絞る。
- 仕様の規則を変える Pull Request は、その規則に依存する記述（付録のテンプレートなど）の更新を同じ Pull Request に含める。

## ライセンス

寄稿は MIT ライセンスの下で受け入れる。Pull Request を出すことで、寄稿を MIT ライセンスで配布することに同意したものとする。
