# 変更履歴

本ファイルは doctrine プラグインのリリースごとの変更を記録する。「なぜ」の根拠は各 ADR に、詳細な差分は git 履歴にある。版番号の正本は `plugin/.claude-plugin/plugin.json`。

## [0.2.1] — 2026-07-22

### 修正

- **リンタが正規の非文書へ ERROR を出し続ける欠陥を解消（ADR-024）。**
  監査（docs-audit）が「非文書／投影」と認めた .md（README・各 skill の SKILL.md・参照散文など実測 37 件）や、統治木の根に到達できない体系外のファイル（ハーネスのメモリなど）に、リンタ（docs-linter）が編集のたび `MISSING_FRONTMATTER` 等の ERROR を出していた。監査（全体を見る）とリンタ（一件を見る）の判定が食い違い、正規状態が恒常的に「要修正」と報告されて、警報の信頼が摩耗し、エージェントを `.txt` 改名などの設計違反の回避へ誘導していた。
  - リンタは判定の前に統治木を探し、次の順で扱う。①統治木が無ければ（体系外）何も出さない。②型付きは従来どおり全点検（統治木の外なら `STRAY_DOCUMENT`）。③型なしで intake に「非文書／投影」と登録されたファイルは schema/frontmatter を飛ばし用語助言のみ（WARN）。④それ以外は従来どおり。

### 追加

- **共有コア `plugin/scripts/_intake.py`。** 分類の記録（`.md-intake`）の読み取りを監査とリンタで一本化。二つの道具が同じコードで同じ記録を読むことで、判定の食い違いを構造的に防ぐ。
- **整合点検 `scripts/consistency-check.py`。** 監査が非文書と認めたファイルにリンタが ERROR を出さないことを機械で確かめる回帰ガード。WATCH-001 に第 5 項として登録。
- **整合点検の横断リマインダ `scripts/consistency-reminder.py`。** セッションを跨ぐカウンタで 10 セッションごとに点検を促す（`.claude/settings.local.json` に配線、`/consistency-check` で手動実行）。

### 内部

- SPEC-007（§制約・§エラー時挙動を条件化）・IMPL-007・TEST-007 を決定に合わせて更新。`test_linter.py` に受入基準①〜④のテストを追加。
- 監査（docs-audit）の intake 読み取りを共有コアへ再配線（挙動は不変）。
- 検証: 全 710 テスト緑・監査 error=0・投影ドリフトなし・整合点検 緑。

## [0.2.0] — 2026-07-07
